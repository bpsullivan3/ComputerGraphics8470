Credit: Based on notes from Sara Mathieson

Rasterizing lines is one of many fundamental topics in a graphics course. The goal of this lab is to implement and test a simple DDA rasterizer. For labs, feel free to compare code and share ideas. Also make sure that others around you are making progress.

Step 1: Download the starter code

Download lab2.html (right-click on this link and then choose "Save As"). Open this file in Sublime (or your preferred editor). This file contains some boiler-plate code and two methods that you'll implement.

Step 2: The drawLine Function

Implement the function drawLine using the DDA rasterization algorithm described in class. Note that this function takes in five arguments in this order: the (x,y) coordinates of the first point, the (x, y) coordinates of the second endpoint, and the line color.

Your method should not use any built-in line drawing functions. Instead, your method should color individual pixels using the setPixel function from your starter code.

This page of HTML Color Names is useful for pre-defined colors.

Step 3: Testing

As you are implementing the drawLine method, make sure to test it out by calling it from the draw method. Your draw method will end up consisting of several calls to your drawLine method. Test it out on lines of various slopes and sizes. Here are some examples: 
    drawLineFast(50, 300, 350, 100, "magenta", 1);  // slope < 1 
    drawLineFast(350, 100, 50, 300, "purple", 1);   // switch points; code should still work
    drawLineFast(200, 80, 100, 380, "red", 1);      // slope > 1     
    drawLineFast(50, 200, 350, 200, "green", 1);    // slope = 0
    drawLineFast(250, 50, 250, 350, "blue", 1);     // slope = ∞

Step 4: Canvas Grid

Now that your drawLine function works, let's draw something on the canvas. We'll start by filling the canvas with a grid pattern (which we may use in the next step).

Comment out the code from the previous step. Inside the function draw, write two independent for loops that create a grid that covers the entire canvas (using calls to the drawLine method). The example below shows a 10x10 lightgrey grid on a 400x400 canvas (so each grid cell is 40x40 pixels). You may choose any grid size or color you want.

Step 6: Extras

(Note: not for extra credit this time, just for more practice with JavaScript functions and HTML canvas.)

  • Experiment with your shape: different pattern, different color, different size.
  • Augment your line drawing method to include a line width argument.
  • Going further: automate your snowflake drawing in a separate method, maybe something that takes as arguments the center position of the snowflake and/or the radius. Use this method to draw multiple snowflakes or randomly positioned snowflakes. You may need a finer grid for this step, or no grid at all.
